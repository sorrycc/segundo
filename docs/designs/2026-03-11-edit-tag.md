# Design: `edit --tag` / `--untag` support

## Summary

Add multi `--tag` and `--untag` flags to the `edit` command, enabling tag-only edits and tag manipulation alongside content edits. Replace the custom arg parser with `yargs-parser`.

## Usage

```bash
# Tag-only edit (no content change)
segundo edit <id> --tag work --tag idea

# Remove tags
segundo edit <id> --untag old --untag draft

# Add and remove tags
segundo edit <id> --tag final --untag draft

# Content + tags
segundo edit <id> "new text" --tag work --untag old

# Batch: --tag/--untag applies to all entries
segundo edit <id1> "text1" <id2> "text2" --tag work
```

## Scope

- Only `edit` gets multi `--tag` and `--untag`.
- Other commands (`add`, `list`, `import`, `export`) keep single `--tag` behavior, reading `argv.tag?.[0]`.

## Decisions

| Decision | Choice |
|----------|--------|
| Untag miss (tag doesn't exist on memory) | Silent no-op |
| Batch edit with --tag/--untag | Applies to all entries |
| Multi-tag syntax | Repeated flags: `--tag a --tag b` |
| Arg parser | Replace custom parser with `yargs-parser` |

## Changes

### 1. Add dependency

```bash
bun add yargs-parser
bun add -d @types/yargs-parser
```

### 2. Replace custom arg parser (`src/cli.ts`, lines 35-75)

Delete `VALUE_FLAGS`, `parseArgs()`, and the manual flag loop. Replace with:

```ts
import yargsParser from "yargs-parser";

const argv = yargsParser(process.argv.slice(2), {
  string: ["brain", "profile", "from", "to", "date", "format"],
  boolean: ["json", "help", "version", "force", "reindex", "batch", "dry-run"],
  array: [{ key: "tag", string: true }, { key: "untag", string: true }],
  alias: { v: "version", h: "help" },
});
```

This gives us:
- `--tag a --tag b` → `argv.tag = ["a", "b"]`
- `--untag x` → `argv.untag = ["x"]`
- `--brain /path` → `argv.brain = "/path"`
- `-v` / `-h` → `argv.version` / `argv.help`
- Positionals → `argv._`

### 3. Update all flag reads (`src/cli.ts`)

| Before | After |
|--------|-------|
| `const { flags, positional } = parseArgs(...)` | `const argv = yargsParser(...)` |
| `flags.version` | `argv.version` |
| `!!flags.json` | `argv.json` |
| `typeof flags.brain === "string" ? flags.brain : undefined` | `argv.brain` |
| `typeof flags.tag === "string" ? flags.tag : undefined` | `argv.tag?.[0]` |
| `flags.limit ? parseInt(flags.limit as string, 10) : 10` | `argv.limit ? Number(argv.limit) : 10` |
| `positional[0]` | `String(argv._[0])` |
| `positional.slice(1)` | `argv._.slice(1).map(String)` |

### 4. New helper (`src/tags.ts`)

```ts
export function removeTags(content: string, tagsToRemove: string[]): string
```

Parses content, filters out matching tags, reconstructs. Silent no-op for tags that don't exist.

### 5. `cmdEdit()` rewrite (`src/cli.ts`, lines 352-398)

- Read tags directly from yargs-parser output:
  ```ts
  const addTags: string[] = argv.tag ?? [];
  const rmTags: string[] = argv.untag ?? [];
  ```
- **Tag-only edit** (id, no content): fetch memory via `getMemory()`, reconstruct full text (body + tag line), apply `appendTags` then `removeTags`, call `editMemory()`.
- **Content + tags**: apply `appendTags` then `removeTags` on new content, call `editMemory()`.
- **Batch**: same logic per entry, `--tag`/`--untag` applies to all.
- **No content, no tags, no untags**: error.
- Import `getMemory` from `./store.js`, `appendTags` from `./tags.js`.

### 6. Help text (`src/cli.ts`, line 141)

```
edit <id> [text]    Edit a memory (--tag/--untag)
```
